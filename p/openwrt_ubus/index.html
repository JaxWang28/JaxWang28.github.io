ubus 概述

ubus 是一个 OpenWRT 中的 RPC 工具，其主要目的就是提供系统级进程间通信。
其核心是 ubusd daemon，提供接口给任何其他 daemon 程序进行注册或发送信息。该接口使用 unix socket 实现，并使用 TLV 格式消息。
ubus 提供了 3 种工具：

  • libubus
  • command-line tool ubus
  • ubus lua module

ubus 通信模型

先看一下我们常见的 client-server 模型。

假设有 n 个程序，那么进程间通信的总 IPC 连接数将达到 n 的累加，这是非常惊人的数量。

使用 ubus IPC 模型。

ubus daemon：作为代理人，负责转发注册。
ubus server object：一些软件或守护进程提供接口，供其他软件调用。Server object 以及其注册的方法可以被 client 调用。
ubus client object：调用者。

ubus 工作模式

Invoke

一对一的通信模式。

Subscribe/notify

一对多的通信模式，以 object 进行分组。

Event Boardcast

多对多的通信模式，通过 event 进行分组。
